## ReplicationController (Old Method)

HA: >> pod1 >> Replicas:1 >> ReplicationController    <br>
SC: >> pod1 >> Replicas:10                            <br>
LB: >> pod1 >>replicas:10 >> service >> pod1-pod10    <br>

Replicas:1 Meaning: 1 pod always will be running      <br>
Replicas:2 Meaning: 2 pod always will be running      <br>


ReplicationController >> check pod labels   <br>
sei dekhe ei name 10 ta pod ache ki nai 



```bash
kubectl api-resources | grep replica
kubectl explain replicationcontroller
```

Create pod defination file
```bash
kubectl run pod1 --image nginx -o yaml --dry-run=client > pod.yaml
vi pod.yaml
```

Create a file name rc.yaml
```bash
---
apiVersion: v1
kind: ReplicationController
metadata:
  name: demorc1
spec:
  replicas: 5
  template:
   metadata:
    name: pod1
    labels:
      app: web
   spec:
    containers:
    - name: doc1
      image: nginx
````
```bash
kubectl apply -f rc.yaml --dry-run=client
kubectl apply -f rc.yaml
```

```bash
kubectl get pods
kubectl get replicationcontrollers
```

Scale
```bash
kubectl scale replicationcontroller demorc1 --replicas 10
```
Now we see pod will 5 to 10

If we want to set no replica
```bash
kubectl scale replicationcontroller demorc1 --replicas 0
```

Edit ReplicationController file
```bash
kubectl edit replicationcontrollers demorc1
```

## ReplicaSet

```bash
kubectl api-resources | grep replica
kubectl explain replicaset
```

<img width="901" height="142" alt="image" src="https://github.com/user-attachments/assets/642d1ae7-74b7-4d90-bdeb-10082e8a70a0" />

Create a file name rs.yaml
```bash
---
apiVersion: apps/v1
kind: ReplicaSet
metadata:
  name: demors1
spec:
  replicas: 5
  template:
    metadata:
      name: pod1
      labels:
        app: web
    spec:
      containers:
       - name: doc1
         image: nginx
  selector:
   matchLabels:
     app: web
```

```bash
kubectl apply -f rs.yaml
kubectl get replicaset
kubectl get pod --show-labels

```

## ReplicationController (Old Method)

HA: >> pod1 >> Replicas:1 >> ReplicationController
SC: >> pod1 >> Replicas:10
LB: >> pod1 >>replicas:10 >> service >> pod1-pod10

Replicas:1 Meaning: 1 pod always will be running
Replicas:2 Meaning: 2 pod always will be running


ReplicationController >> check pod labels 
sei dekhe ei name 10 ta pod ache ki nai 



```bash
kubectl api-resources | grep replica
kubectl explain replicationcontroller
```

Create pod defination file
```bash
kubectl run pod1 --image nginx -o yaml --dry-run=client > pod.yaml
vi pod.yaml
```

Create a file name rc.yaml
```bash
---
apiVersion: v1
kind: ReplicationController
metadata:
  name: demorc1
spec:
  replicas: 10
  template:
   metadata:
    name: pod1
    labels:
      app: web
   spec:
    containers:
    - name: doc1
      image: nginx
````
```bash
kubectl apply -f rc.yaml --dry-run=client
kubectl apply -f rc.yaml
```

```bash
kubectl get pods
kubectl get replicationcontrollers
```

If I delete the pod, it will create autometically
```bash
kubectl delete pod demorc1-qvbrb
kubectl get pods
```

Scale
```bash
kubectl scale replicationcontroller demorc1 --replicas 5
```
Now we see pod will 10 to 5

If we want to set no replica
```bash
kubectl scale replicationcontroller demorc1 --replicas 0
```

Edit ReplicationController file
```bash
kubectl edit replicationcontrollers demorc1
```
